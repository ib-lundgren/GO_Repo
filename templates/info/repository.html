Introducing the Game Object Repository

The game object repository will allow anyone to reuse, edit and supply game objects. A game object might be a character model, a quest with objectives, a weapon with attachable accessories, a vehicle with special controls. We will supply a flexible way of creating vastly different and content rich games.

The repository will feature a restful API. This API will be made available to the public and can be used to create entire game creators, object modelling tools, full featured games, rich fan sites to name a few. In our first release of the API we will supply a basic game object editor and a demo game created using our game objects and a Javascript Canvas game engine.


Game object categories


    No Interaction Environment

    Environment that has a no effect on the character and is mainly used to create a richer gaming experience.

    Possible objects in this category include

    • Background
    • Foreground
    • Weather effects

    Passive Interaction Environment

    Environment that has a passive effect on characters and vehicles. Imagine your character walking into a swamp, then he might start to walk slower and this is the passive effect. Another scenario would be a character standing on ground, the ground pushes him up with a force equal to the gravity. If the ground was to be removed, he would fall.

    Possible objects in this category include

    • Dirt
    • Fire
    • Swamp
    • Power up

    Active Interaction Environment

    Environment that can be interacted with. A good example is Mario going down through pipes by pressing the "Down" button. It could be a ticking bomb that could be defused or water you can swim in.

    Possible objects in this category include

    • Water
    • Door
    • Ladder
    • Treasure chest

    Character

    Characters can have a multitude of shapes and forms but most will likely have a few common features. They might include a graphical model, a level, lists of equipment, skills, achievements, ranks, reputation, score and which teams it is a member in.

    Possible objects in this category include

    • Adventurer (Roleplaying)
    • Rifleman (Strategy)
    • Terrorist (Shooter)
    • Monkey (Arcade)

    Vehicle

    Vehicles that can be controlled by players. They could have properties like speed, acceleration, armor, fuel level, gears and a list of drivers and passengers.

    Possible objects in this category include

    • Horse
    • Tank
    • Jetpack
    • Snowboard
    • Elevator

    Equipment

    Equipment used by characters. Might include an AK-47 with detachable scope, an inventory bag, currency, potions and funky mushrooms.

    Possible objects in this category include

    • Weapon
    • Ammunition
    • Armor
    • Consumable
    • Inventory

    Mission / Quest

    The goal of the game is usually achieved through a series of missions or quests. These usually have a objectives, information, rewards and pre-conditions to be able to take it.

    Possible objects in this category include

    • PickUp20000000DeerAntlers
    • KillEnemyTeam
    • DefuseBomb

    Teams

    Teams are a group of characters or other teams. Using teams you can construct anything from a dungeon group, to a football team to a full faction or guild.

    Possible objects in this category include

    • Terrorists
    • Alliance
    • TheSpanishBobsledgeTeam

    Map / Game Level

    A map is a collection of game objects. Maps usually make up the core of the game and the entire game can be seen as a collection of maps. Each map might have a lists containing environments, vehicles, characters, teams and mission objects.

    Possible objects in this category include

    • SideScrollerLevel
    • Azeroth
    • Brands Hatch F1 Circuit

    Event

    An event can be practically anything that happens during a game. Game developers might want to save statistics of what achivements players get, or they might want to save every little event that happens to allow the entire game to be replayed later.

    Possible objects in this category include

    • Headshot
    • MissionComplete
    • PlayerDisconnected

    Player

    A player represents the actual human player. In an RPG the player might control one or more characters, in a shooter he might choose a new character on each game, in tetris he doesn't have a character at all. Player can also be used to track statistics like total time played, accuracy and ladder rank.

    Possible objects in this category include

    • RPG Player
    • FPS Player

    NPC

    NPCs are basically computer controlled players. They are mainly found in Maps/Levels but can also act very similar to a normal player in the case of bots.

    Possible objects in this category include

    • QuestGiver
    • Boss
    • Bot

    Game

    A game is the collection of game objects that make up a game. This can include lists of levels, players, npcs. It might also contain high scores and restrictions on maximum amount of players.

    Possible objects in this category include

    • Angry Birds
    • Battlefield 3
    • Tetris

    Modifiers

    Possible objects in this category include

    • IncreaseWeaponDamage (Equipment)
    • IncreaseMountSpeed (Vehicle)
    • ChangeLocation (Character)
