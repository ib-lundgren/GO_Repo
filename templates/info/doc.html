Using the Game Object Repository API

Game objects can be created using the Game Object Editor, then you only need to login using your Google account and edit away!

If you want to interact with the API directly, you do so using an API key. You can obtain an API key by logging in with your Google account and requesting one on our API key request page.

An overview of the API

We currently support GOs in the following categories; Graphic, Environment, Character, Equipment, Level, Mission, Vehicle, Interaction, ....

If an invalid category or non existing object is requested, an error message (json) will be returned (HTTP 404).

GET - /api/<category>

Returns a json list of objects in <category> (HTTP 200).

curl -X "GET" https://go-repo.appspot.com/api/Environment/
POST - /api/<category>

Creates a new object of <category> type, returns a json representation of the object (HTTP 201) or an error message if the form was invalid (HTTP 400).

curl -X "POST" -d "title=example&description=this is a test&trigger=Enter&action=hello world&api_key=sdf7yf" https:/go-repo.appspot.com/api/Interaction/
GET - /api/<category>/<id>/

Returns a json representation of the object with id=<id> in <category> (HTTP 200). 

curl -X "GET" https://go-repo.appspot.com/api/Interaction/401/
PUT - /api/<category>/<id>/

Updates an existing object, parameters and error messages are identical to creating a new object.

curl -X "PUT" -d "...." https://go-repo.appspot.com/api/Interaction/39487234/
DELETE - /api/<category>/<id>/

Deletes an object, this will only succeed if the creator of the object is the one requesting deletion. Returns a delete successfull message (HTTP 200) or an unauthorized message (HTTP 401).

curl -X "DELETE" -d "api_key=23762349" https://go-repo.appspot.com/api/Mission/2348933/
POST - /api/<category>/<id>/

Adds references to other objects to an object. For example, a character can have a range of different equipment objects. Returns a representation of the updated object (HTTP 200) or an error message if the update fails (HTTP 400).

curl -X "POST" -d "equipment=234234234, 346346346, 346666zsdf" https://go-repo.appspot.com/api/Character/39857234/

Installing a clone of GO-Repo on appengine


Install the Python Appengine SDK and make sure appcfg.py and dev_appserver.py is on your path and marked as executable. You will also need Python and Git.


Get the source code for the website 

git clone git://github.com/ib-lundgren/GO_Repo.git

Change the app.yaml name field to match your appspot name. Upload to appspot using appcfg.py or try it out locally with dev_appserver.py 


$ dev_appserver.py GO_Repo/
$ appcfg.py update GO_Repo/
